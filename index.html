HTML

  • Element: what is inside an opening tag and a closing tag.
  • Tag: what is inside the angle brackets.
  • Brackets are less than and greater than signs in HTML.
  • "head" holds information for site and browser, contains meta (description, keywords, author, viewport) and title.
  • "body" holds all the information the user can see and interact with, like this information being read now.
  • "ul" unordered list, for bullet points like this doc.
  • "ol" ordered list, for numbered lists.
  • "h1" for main title, the higher "h(x)" the smaller the title text.
  • "br /" in brackets for break in lines.

CSS

  • 3 different ways to style in CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline CSS: for syling a single element directly in the HTML file. Example: adding style="color:red;">Heading that makes the heading red.
  • Internal style sheet: contains rules for the webpage in the head section of the HTML file. Example: (in brackets) style, tab in, h1 { color :red; }
  • External style sheet: most common and easier to change, implants a link in file. Example brac head tab brac link rel="stylesheet" href="./assets/style.css" / brac

Git

  • git status: checks what branch we're in.
  • git checkout main: switch back to main branch.
  • git checkout -b branch-name: creates new branch or switches to it.
  • git add -A: to commit all files in working branch.
  • git commit -m "x": to associate a message with the implemented commit.
  • git pull origin main: to update any changes to main before inserting your branch.
  • git push origin feature/branch-name: to push all the changes in branchg to main.

Command Terminal commands:

  • ls = list of files.
  • cd = change directories (restores default I.e. no directory selected).
  • cd (x) = change to x directory.
  • pwd = print working directory I.e. let me know wha directory we're in.
  • mkdir (x) = Make directory titled x
  • touch (x) = create file x.
  • open (x) = open x.
  • code . = open VScode.
